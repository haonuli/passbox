# Passbox 全面功能优化方案

> 本文档为阶段 1-4（需求定义 + 技术方案 + 任务拆分 + 测试策略）的完整产出，经用户确认后作为开发执行的依据。

---

## 阶段 1：需求定义与验收标准

### 1.1 登录/注册页面重构

**需求**：将 `/login`、`/register`、`/recover` 三个认证页面改造为左右分栏布局。

- 左侧（40%）：品牌展示区，包含产品 Logo、核心价值主张、安全特性说明、装饰性视觉元素
- 右侧（60%）：表单区域，包含原有表单逻辑
- 移动端：隐藏左侧品牌区，仅展示表单
- 暗色模式适配

**验收标准 (AC)**：
- AC1: 三个认证页面均采用左右分栏布局
- AC2: 左侧品牌区在 `md` 以下屏幕隐藏，右侧表单居中
- AC3: 暗色模式下视觉表现正常
- AC4: 表单功能完全不变（登录/注册/恢复流程不受影响）
- AC5: 左侧品牌区包含至少 3 个安全特性说明

### 1.2 输入验证系统增强

**需求**：建立集中化的验证规则库，增强所有表单的输入校验。

- 创建 `src/lib/validations.ts`，集中管理所有验证规则
- 增强密码强度规则（增加特殊字符建议，但不强制）
- 为条目表单中的字段添加类型化验证（URL、邮箱、手机号、IP 地址、信用卡号等）
- 统一错误提示文案风格

**验收标准 (AC)**：
- AC6: 所有验证规则集中在 `src/lib/validations.ts` 中，可复用
- AC7: URL 字段验证格式合法性（不要求可访问）
- AC8: 邮箱字段使用统一规则
- AC9: 手机号字段支持中国大陆手机号格式
- AC10: IP 地址字段验证 IPv4 格式
- AC11: 信用卡号字段验证 13-19 位数字
- AC12: 密码验证规则保持现有安全性（12位+大小写+数字），提示文案更友好

### 1.3 三栏式布局重构

**需求**：将密码库主页面从「列表 -> 跳转详情页」改为「列表 + 详情」并排展示的三栏布局。

- 左栏：导航菜单（现有 Sidebar）
- 中栏：条目列表（现有 VaultView）
- 右栏：选中条目的详情展示（内嵌 ItemDetail，不再跳转独立页面）
- 点击列表项 -> 右侧展示详情，不离开当前页面
- 新建/编辑条目仍使用独立页面（表单复杂度高）
- 移动端：列表和详情切换显示（点击列表项全屏展示详情，返回回到列表）

**验收标准 (AC)**：
- AC13: 密码库页面呈现三栏布局（导航 + 列表 + 详情）
- AC14: 点击列表条目，右侧详情区同步更新
- AC15: 详情区域有空状态（未选中条目时的引导提示）
- AC16: 删除条目后详情区回到空状态
- AC17: 移动端列表/详情切换正常
- AC18: 其他页面（安全中心、生成器、设置）保持现有布局

### 1.4 额外优化项

| # | 优化点 | 说明 |
|---|--------|------|
| O1 | 清理废弃代码 | 删除 register 目录下未使用的 `strength-indicator.tsx`（已被 common 版替代） |
| O2 | 统一密码输入组件 | 登录表单中使用手动 Eye/EyeOff，应改用已有的 `PasswordInput` 组件 |
| O3 | 落地页优化 | 首页增加功能亮点展示，使用 next/link 替代 `<a>` 标签 |
| O4 | 搜索体验优化 | 搜索框增加快捷键提示（⌘K），提升专业感 |
| O5 | 加载状态骨架屏 | 列表加载时展示骨架屏替代单纯 Spinner |

## 阶段 2：技术方案设计

### 2.1 认证页面分栏布局

**方案**：创建共享布局组件

```
src/components/auth/
├── auth-split-layout.tsx    # 左右分栏容器
└── auth-brand-panel.tsx     # 左侧品牌展示面板
```

- `AuthSplitLayout` 接收 `children`（右侧表单），渲染左右分栏
- `AuthBrandPanel` 渲染品牌信息、安全特性列表、装饰性渐变背景
- 三个认证页面（login/register/recover）的 `page.tsx` 使用 `AuthSplitLayout` 包裹
- 左侧面板在移动端通过 `hidden md:flex` 隐藏

### 2.2 验证规则集中化

**方案**：创建 `src/lib/validations.ts`

导出可复用的 zod schema 和正则表达式：
- `emailSchema` — 邮箱验证
- `passwordSchema` — 主密码验证（12位+大小写+数字）
- `urlSchema` — URL 格式验证
- `phoneSchema` — 中国大陆手机号验证
- `ipv4Schema` — IPv4 地址验证
- `creditCardSchema` — 信用卡号验证（13-19位数字）
- `recoveryCodeSchema` — 恢复码格式验证

各表单组件从 `validations.ts` 导入规则，消除重复定义。条目表单的 `buildSchema()` 函数根据字段名匹配验证规则。

### 2.3 三栏布局实现

**方案**：重构 vault 页面为三栏联动视图

```
src/app/(app)/vault/_components/
├── vault-view.tsx          # 主视图（改为三栏容器）
├── vault-toolbar.tsx       # 列表工具栏（保留）
├── item-row.tsx            # 列表行（增加选中态）
├── item-detail-panel.tsx   # 右侧详情面板（新，封装 ItemDetail 逻辑）
```

- `VaultView` 内部管理 `selectedItemId` 状态
- 点击 `ItemRow` 设置 `selectedItemId`，右侧面板渲染对应详情
- `ItemDetailPanel` 复用现有 `ItemDetail` 组件的核心逻辑
- 详情面板增加空状态引导
- `/items/[id]` 路由保留（直接跳转时自动选中对应条目）
- 移动端通过 `selectedItemId` 控制列表/详情切换

### 2.4 其他优化技术方案

- **O2 统一密码输入**：`login-form.tsx` 中移除手动 Eye/EyeOff，替换为 `PasswordInput`
- **O3 落地页**：`page.tsx` 中 `<a>` 替换为 `<Link>`，增加功能亮点区域
- **O5 骨架屏**：创建 `vault-skeleton.tsx` 列表骨架组件
